Project overview:

- Fantasy overworld hex map web app.
- Leaflet-based map with parchment UI aesthetic.
- The “Codex” acts as an in-world atlas/database browser for:
  - Regions
  - POIs
  - NPCs
  - Hexes
- Heavy emphasis on immersive parchment UI, readable lore presentation, and app-like UX.

Current architecture/state:

- Codex uses shared rendering helpers:

  - applyFilters()
  - shared sort helpers
  - shared label builders
- Multi-filter AND logic implemented.
- Dynamic filter-field selector system implemented.
- Shared list-page architecture implemented.
- Shared detail-page architecture implemented.
- Search system implemented with grouped result sections:
  - Hexes
  - Regions
  - POIs
  - NPCs
- Search caching/back-navigation behavior implemented.
- Search now also returns:
  - hexes containing matching NPCs/POIs
  - hexes inside matching regions

- NPC/POI detail pages redesigned with:
  - portrait slot area

- Mobile long-press easter egg implemented for Codex button.

Important design direction:

- Desktop and mobile are intentionally diverging somewhat in layout.
- Desktop favors immersive “atlas spread” layouts.
- Mobile should feel more app-like and space-efficient.
- Avoid breaking desktop UX while improving mobile.

Important implementation philosophy:

- Prefer surgical modifications over large rewrites.
- Always inspect the existing live function/class before replacing.
- Avoid duplicate IDs or duplicated control markup.
- Avoid abstract assumptions about current code structure.

Current notable mobile implementation:

- Filter/sort controls on mobile are now a modal/popover opened via a “Filter & Sort” button.
- The same controls shell is reused rather than duplicating controls.
